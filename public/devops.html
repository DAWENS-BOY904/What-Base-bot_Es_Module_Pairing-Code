Module 1 — Git & Source Control

Essential commands, workflows, branching strategies, and a lab to practice merging and rebasing safely.

1.1 — Core Git Commands

Clone, add, commit, push, pull, branch, merge, rebase, stash, log.

// Initialize a repository and make the first commit
git init my-project
cd my-project
echo "# My Project" > README.md
git add README.md
git commit -m "chore: initial commit"
git branch -M main
git remote add origin git@github.com:YOURUSER/my-project.git
git push -u origin main
Tip: use git status and git log --oneline --graph --decorate frequently.

1.2 — Branching & Workflows

Feature branches, GitFlow basics, pull request etiquette.

// Create a feature branch and push
git checkout -b feat/login-form
# work...
git add .
git commit -m "feat(login): add login form"
git push -u origin feat/login-form

1.3 — Lab: Safe Merge vs Rebase

Follow steps to practice resolving conflicts, using rebase to keep history clean, and merging for release branches.

  1. Make a repo with two branches (main & feature).
  2. Introduce conflicting changes on both branches and practice git merge.
  3. Reset and practice git rebase main on the feature branch and resolve conflicts.
// Example conflict resolution workflow
git checkout feat/login-form
git fetch origin
git rebase origin/main
# resolve conflicts in editor, then:
git add 
git rebase --continue
git push -f origin feat/login-form

Module 3 — Docker: Containers & Images

Learn to containerize apps, build multi-stage images, and run containers locally and in CI.

3.1 — Dockerfile Basics

# Dockerfile - multi-stage build for Node app
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --production=false
COPY . .
RUN npm run build

FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY package*.json ./
RUN npm ci --production=true
CMD ["node", "dist/index.js"]

3.2 — Useful Commands

# Build image
docker build -t myapp:latest .

# Run container with environment and port mapping
docker run -d --name myapp -p 8080:8080 -e NODE_ENV=production myapp:latest

# List images and prune
docker images
docker system prune -a

Module 4 — CI/CD Pipelines

Automate builds, tests, and deployments. We'll provide GitHub Actions and GitLab CI examples.

4.1 — GitHub Actions (Simple Pipeline)

# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: 18
      - name: Install
        run: npm ci
      - name: Run tests
        run: npm test
      - name: Build
        run: npm run build
      - name: Publish Docker
        uses: docker/build-push-action@v4
        with:
          push: false
          tags: user/myapp:${{ github.sha }}
          load: true
          file: ./Dockerfile
          context: .
          platforms: linux/amd64
          cache-from: type=gha
          cache-to: type=gha,mode=max
          build-args: |
            NODE_ENV=production
          secrets: |
            DOCKERHUB_USERNAME:${{ secrets.DOCKERHUB_USERNAME }}
            DOCKERHUB_TOKEN:${{ secrets.DOCKERHUB_TOKEN }}
          env:
            REGISTRY: docker.io
          # Note: configure registry and push in a production setup

4.2 — GitLab CI Example

# .gitlab-ci.yml

4.3 — Blue/Green & Canary Deployments

Strategies to reduce risk when deploying to production — basics and when to use each strategy.

# Canary approach (conceptual)
# 1. Deploy new version to small subset of traffic (10%)
# 2. Monitor metrics (errors, latency)
# 3. If stable, increase traffic fraction progressively
# 4. If failure, rollback to previous stable version

Module 8 — Kubernetes Basics

Pods, Deployments, Services, ConfigMaps, and simple manifest examples.

8.1 — Deployment + Service

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 2
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: user/myapp:latest
        ports:
        - containerPort: 8080

---
apiVersion: v1
kind: Service
metadata:
  name: myapp-svc
spec:
  type: ClusterIP
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 8080

CI: Test matrix example

# GitHub Actions matrix example
name: CI Matrix
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [16, 18, 20]
        os: [ubuntu-latest, windows-latest]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: node-version: ${{ matrix.node-version }}
      - run: npm ci
      - run: npm test

Debugging & Common Pitfalls

Real-world issues and how to quickly troubleshoot them.

Docker Container won't start

  1. Check container logs: docker logs CONTAINER
  2. Run an interactive shell: docker run -it --entrypoint /bin/sh IMAGE
  3. Check healthcheck failures

CI job fails on unknown dependency

Pin dependency versions and cache installs in CI.

Kubernetes CrashLoopBackOff

# Inspect pod
kubectl describe pod mypod
kubectl logs mypod -c containername
# If crash due to env, check ConfigMap and Secrets
kubectl get secrets

Bash: Wait for service (retry)

#!/usr/bin/env bash
set -e
for i in {1..30}; do
  if curl -sSf http://localhost:8080/health >/dev/null; then
    echo "Service up"
    exit 0
  fi
  echo "Waiting for service... ($i)"
  sleep 2
done
echo "Service did not start" >&2
exit 1
